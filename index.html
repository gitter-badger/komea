Komea is an Open Source Framework that...

Multiple connectors.

Collect and store data from your software factory

  • Git, SVN, CVS, Perforce
  • Redmine, Bugzilla, Jira
  • Testlink
  • Jenkins
  • SonarQube
  • LDAP, Active Directory

Extensible timed event model.

All the events are stored into an history NoSql database. Database is schema less and can be easily use to store your own event data structure.

Extensible hierarchical organization model.

Organization of your sofware factory and development process are stored into an NoSql organizational database.

  • Hierarchical. For example, a group contains some developers.
  • Graph database. For example, developers are involved into multiple applications.
  • Extensible to fit your needs

Query language.

Events and organization databases can be queried trough some API to build some advanced metrics on your whole software factory.

  • SQL like syntax
  • Predicates and filters.
  • Results aggregation (sum, average, min, max ,etc.).
  • Results export

Lightweight Business Intelligence module.

Komea contains a basic BI module that can be used to monitor your software factory.

  • Create metrics that will be scheduled periodically.
  • Store timed results of metrics in a database
  • Metrics graph export (bar chart, time chart, bubble chart)

RESTful API.

All databases can queried and updated trough a REST API.

  • Events
  • Organization
  • Metrics
